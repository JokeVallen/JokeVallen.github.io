AssetBundle 加载复合资源

复合资源是指类似于被切片为多个 Sprite 的 Texture2D 或者 FBX 模型（附带 Mesh、纹理）等在导入时划分为主资源和内部资源的资产对象，例如被切片的 Texture2D，文件本身是主资源，在 Unity 中又被切片为多个 Sprite 资源，它们本质上是存在一个父子关系，所以这里称为复合资源，不同于引用图片或音频的预制体，那是组合关系，不是派生关系。

AssetBundle 从本地文件、内存和数据流加载

从本地文件加载

这里只做简单说明，具体使用会在后续篇幅阐述，多数加载方法都具备同步和异步版本，这里我们通常使用传入 path 参数即可，除非涉及 CRC 校验或者从整合包中加载分包才会涉及另外两个参数。

从内存创建

这是一种兼容特殊情景的补充式 API，由于涉及到拷贝，所以对于大体量 AssetBundle 会带来明显的内存压力，所以这不是主流和推荐的加载方式，只是为了应对特殊情景，例如解密后的 AssetBundle 字节数据或者从自定义资源容器中提取 AssetBundle，本质上就是从字节数据创建 AssetBundle。

从数据流加载

这是 AssetBundle 流式加载的 API，关于流式加载的使用场景不作阐述，但这通常是多数用于资源加载的工具库所必备的。

从网络下载 AssetBundle

UnityWebRequestAssetBundle

这是 Unity 官方提供的一个创建特定优化版本的 UnityWebRequest 的工厂类，专门为 AssetBundle 资源的下载提供快速创建请求的入口。

DownloadHandlerAssetBundle

这是 AssetBundle 网络下载特供的下载处理器，继承自 DownloadHandler，内部将下载数据流式传输到 Unity 的 AssetBundle 解压和解码系统，以提供高效的下载和处理方式。

C# 原生网络库相关 API

将 AssetBundle 视为普通二进制数据，使用 HttpClient 或 WebClient 以及其它网络传输下载相关 API 去下载到本地，然后从本地进行加载。不过这种方式需要自己封装缓存、断点续传等功能，对于工具库或者项目框架开发者而言可能会涉及这种原生方式，因为这代表着灵活和自由度。

通过 Addressables 或 YooAsset 等插件

这些插件都会封装相关原生 API 以及进行某些优化，提供更具易用性的 API，但往往这意味着牺牲部分灵活性和自由度，这是不可避免的，具体应结合项目需求衡量选择什么解决方案。
