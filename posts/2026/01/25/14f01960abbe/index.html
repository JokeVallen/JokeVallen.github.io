基于Hexo、Butterfly和Github搭建个人博客

更新于2026-01-25|通用知识
|浏览量:

本文中部分工具的下载和链接需要网络代理才能正常访问

前言

  • Node.js： 一个开源的、跨平台的 JavaScript 运行时环境。它的核心特点在于使用 V8 引擎，使得开发者能够使用 JavaScript 在服务器端编写和运行高性能、可扩展的网络应用程序。它采用事件驱动和非阻塞 I/O 模型，特别适合处理数据密集型的实时应用。因此，Node.js 极大地统一了Web开发的编程语言，让前端与后端都可以使用JavaScript，是现代全栈开发的重要基础。 ➡ 官网直达

  • Hexo： 一个基于Node.js的快速、简洁且高效的静态博客框架。它允许用户使用Markdown或其他标记语言编写文章，并生成静态网页文件。Hexo具有丰富的主题和插件生态系统，支持一键部署到GitHub Pages等平台，适合需要轻量级、可定制化博客的用户。 ➡ 官网直达

  • Butterfly： 一款为Hexo博客框架设计的功能全面、设计优雅的现代化主题。它以简洁大方的卡片式设计和响应式布局著称，并整合了大量实用功能，包括但不限于多种评论系统（如Disqus、Gitalk）、深色模式、Pjax无刷新加载、文章目录、数学公式支持（MathJax）、代码块高亮与复制，以及丰富的页面类型（如相册、友链、音乐播放器）。主题高度可定制，拥有活跃的开发者社区持续更新优化，是追求美观与功能性兼备的个人博客用户的流行选择。 ➡ 官网直达

  • GitHub： 全球领先的、基于Git版本控制系统的代码托管与开发者协作平台。它为个人和团队提供项目托管、分布式版本控制、代码审查（Pull Request）、任务管理（Issues）等功能，形成了一个规模巨大的开源社区。2025年时，该平台已拥有超过1.5亿开发者和4.2亿个代码仓库。GitHub于2008年创立，于2018年被微软收购，并持续演进，推出了GitHub Copilot等AI编程工具，致力于成为服务于整个开发生命周期的综合性平台。 ➡ 官网直达

  • Visual Studio Code： 一款由微软开发的免费、开源、跨平台的现代化代码编辑器，它通过丰富的扩展生态系统和对内置Git、智能代码补全、调试等功能的强力集成，成为了一个高度可定制且功能强大的集成开发环境。 ➡ 官网直达

环境安装

  • Node.js： 在官网下载并安装。

  • Hexo与Butterfly： 建议通过Node.js安装，通过控制台的npm指令，具体CLI请查看各自官网或文档。

  • GitHub： 在电脑上安装Git，则可以通过CLI完成对GitHub项目仓库的多样操作，如果希望通过可视化界面操作，可以下载官网的GitHub Desktop软件。

  • IDE： 可以考虑使用Visual Studio Code，比较轻量，同时具有丰富的插件生态，能够支持多种格式的文本的编写，例如Markdown可以通过代码片段、markdownlint插件以及内置的Markdown预览窗口来实现快捷编写、格式矫正、语法检查和实时预览。

搭建流程

搭建流程相关描述基于环境安装已完成的前提，请不要看一段而操作一段，应该看完所有流程内容后再进行实操，因为这里的内容书写顺序并不代表正确的实操顺序。

Hexo

  • 通过Hexo的CLI创建本地博客项目，并根据官方文档完成相关配置（项目根目录下的配置文件 _config.yml，可参考下文中的 参考配置或代码 部分）。

  • 配置博客项目根目录的 .gitignore 文件（可参考下文中的 参考配置或代码 部分），因为通常我们会把博客项目放置在私有的仓库中，而部署则另外通过一个公开的仓库来完成，这里的 .gitignore 文件影响的是私有仓库，项目中并非所有内容都需要上传到仓库中去备份。

  • 在博客项目根目录的 node_modules 目录下找到博客主题目录下的 _config.yml 文件，复制一份并改名为 _config.主题名.yml，例如 _config.butterfly.yml（如果使用的是Butterfly主题，可参考下文中的 参考配置或代码 部分），根据主题官方文档完成对主题的配置文件的设置。

  • 在控制台通过CLI hexo new post <文章名> 来根据模板生成一篇博文，它通常位于 项目根目录/source/_posts 目录下，打开IDE编写博文，保存即可。值得注意的是，我们几乎所有资源都应该放在source目录或其子目录下。如下所示：

  • source

    • css
      • custom.css
    • js
      • custom.js
    • images
      • custom.png

    ……

  • 在控制台通过CLI hexo clean & hexo g & hexo s 来清理旧的生成文件、重新生成然后开启本地服务器浏览，Ctrl+鼠标左键 点击控制台的本地服务器链接即可在浏览器中查看博客网站，修改博文内容和改样式文件内容等通常不需要重新执行上述CLI，hexo提供了监视功能，会进行刷新，改动后我们只需要在浏览器刷新页面即可看到修改效果。

  • 当你通过本地服务器+浏览器查看博客网站，认为效果理想后，即可在控制台通过CLI hexo d 将博客网站一键部署到GitHub，从而让我们的博客网站暴露在公网中，能够被外部访问。

GitHub

  • 我们需要创建两个仓库，一个私有仓库用于上传本地博客项目，作为开发备份，另一个公开仓库作为博客网站的访问接口，这是更安全的方式，防止你将所有内容暴露在公网中，导致隐私泄露。

  • 我们可能需要在 GitHub -> Settings -> Developer settings -> OAuth Apps 中创建一个博客应用，使得公开仓库作为我们博客网站某些功能或内容的存储媒介，例如Gitalk插件则需要我们这个流程来实现访客登录GitHub账户并在网站中留下评论内容，而这些内容则存储在我们的公开仓库中，博客应用中的 Client ID 、 Client Secret 等字段则需要配置到我们的配置文件中，使得我们的插件能够正常工作。具体可以浏览各类插件的官方文档，会有具体的引导。
