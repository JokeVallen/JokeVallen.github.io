UGUI的布局系统（一）

更新于2026-02-08|Unity开发
|浏览量:

前言

本系列内容参考自官方文档，并结合具体实践进行讲解，帮助开发者更好地理解 UGUI 的布局系统。学习 UGUI 的布局系统，我们先从官方已有的布局组件开始学习，本文将逐一讲解它们的代码和布局效果。在项目 UI 开发的需求中，布局是常见的需求，当你注意到它们的时候意味着你的项目需要适配不同分辨率的设备。

事实上 UGUI 官方提供的布局组件能适应大部分布局需求，因为布局组件可以组合使用实现预期效果，例如父元素需要根据子元素的大小变化而自适应或者子元素在父元素有限的范围中灵活分布等等。

本文先从 RectTransform 、Anchors 、 Pivot 三个概念开始讲解。

RectTransform

这是 UGUI 所有 UI 元素都离不开的核心组件，它承担着 UI 元素的位置、旋转、大小的控制，所有布局组件的计算都离不开它。要学习它离不开几个两个重要概念：Anchors 和 Pivot。

Anchors

通常被称为锚点，顾名思义，它作为 UI 元素位置和大小控制的参考点，当我们需要 UI 元素始终保持在父元素的相对位置时，就可以将锚点锚定在父元素矩形的任意位置，锚点值是相对父元素空间的，最小值为0，最大值为1，左下角作为参考坐标系的原点位置，右上角作为最大值的位置。锚点分为两组数据分别是 Min 和 Max，其中各自又包括 X 和 Y，对应水平位置和竖直位置，所以一共由 MinX、MinY、MaxX、MaxY 四项组成。MinX 和 MaxX 分别作为元素左和右的参考值，MinY 和 MaxY 则分别作为元素下和上的参考值。

例如 Min(0,0) 和 Max(0,0) 则代表基于父元素矩形的左下角为 UI 元素四侧的参考点，那么只有当父元素的左侧或下侧有所变化时才会影响到该元素的位置。如下图所示：

例如 Min(0,0) 和 Max(1,0) 则代表基于父元素矩形的左下角、右下角作为参考点，那么只有当父元素的左侧或右侧或下侧有所变化时才会影响到该元素的位置或宽度。如下图所示：

当我们需要让 UI 元素在父元素的矩形空间中保持相对位置和大小时，可以通过修改锚点值来满足需求，官方也提供锚点预设，如下图：

Pivot

通常被称为轴心，它影响着 UI 元素旋转、大小和缩放的效果，在编辑锚点时，应该将轴心模式从设置为Pivot，这样我们对轴心值的修改才能在 Scene 视图中看见轴心位置。如图所示：

轴心被视为 UI 元素自身的中心，注意这里的中心不是说图形的对称中心，它的值可以是小于-1或大于1的，因为 UI 元素的中心也可以在其矩形范围之外。例如我们将轴心值设置为 (-1,0.5)，然后调整 Rotation Z 的值就能围绕元素矩形范围外的点进行旋转，如图所示：

既然轴心被视为 UI 元素自身的中心，那么它也会影响 PosX 、PosY、PosZ 的值，因为根据锚点计算相对坐标需要参考 UI 元素的中心点，上述三个值可以理解为 UI 元素的中心点相对于锚点坐标系的坐标值。轴心也会影响 UI 元素的缩放参考，例如我们将轴心值设置为 (0,0)，然后调整 Scale X 和 Scale Y 的值就能基于该轴心点进行缩放，如图所示：

最后我们回到 RectTransform 组件，上述两点作为 RectTransform 的核心概念，也是我们学习布局系统必须掌握的基础知识，只有清除了它们的功能和原理才能帮助我们更好地理解布局系统的运作。

总结

本文主要讲述了 RectTransform 、Anchors 、Pivot 三个重要概念，下一篇文章我们针对 UGUI 布局系统的组成内容和工作原理进行讲解。
