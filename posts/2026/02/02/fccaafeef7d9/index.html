Unity自定义项目模板

更新于2026-04-29|通用知识
|浏览量:

不同版本的 Unity 编辑器对于项目模板的自定义可能存在区别，本文以博主个人使用的版本为主，其它版本看情况后续进行补充。

前言

自定义项目模板尤其适用于团队开发，特别是对于存在一套成熟框架的情况，可以在项目模板中直接引入框架，同时自定义项目模板也能让刚新建的项目处于干净简洁的状态，可以根据项目具体需求去手动引入，也可以在自定义项目模板的时候去配置相关文件来自动引入。

值得注意的是，自定义项目模板更适合对 Unity 内置包不陌生的开发者，特别是需要高度自定义的情况，如果不熟悉内置包，可能存在你需要开发 2D 项目，却不知道应该引入哪些依赖，最好的方式是基于 Unity 的现有核心模板去修改，而不是在卸载所有依赖后再逐一添加。

模板创建流程

新建模板项目

通过 UnityHub + Unity 的内置模板创建一个模板项目，名称没有限制，也可以按照内置模板格式，例如 com.unity.template.custom.empty。

完成模板项目的自定义配置

新建好的模板项目可以进行包括但不限于以下自定义配置：

  • 项目设置（Project Settings）：例如团队名称（Company Name）、开头动画（Splash）等。

  • 依赖包设置（Package Manager）：对于基于该模板创建的项目应该具备的共同依赖包，例如 Visual Studio Code Editor 等。通常我们可以借助 Unity 的 Package Manager 去管理依赖包，对于需要高度自定义依赖包的开发者也可以手动去修改 Packages 目录下的 manifest.json 文件，它负责管理项目的依赖包，没有必要去修改 packages-lock.json，它会根据你的 manifest.json 文件重新生成，但是修改了 manifest.json 文件后应该触发 Unity 的重新编译，以保证packages-lock.json 同步更新。

  • 项目基础目录（Project窗口）：例如包含 Plugins 、 Resources 、 Scenes 、StreamingAssets 、 Scripts 等目录。

  • 项目基础内容（Project窗口）：例如项目基本框架、常用编辑器工具等。

整理模板结构及内容

这一步是基于你前置流程完成的情况下，打开项目根目录，然后你通常能看到以下结构：

项目根目录结构

  • 只保留 Assets 、 Packages 、 ProjectSettings 三个目录，其它内容都删除掉或者放到备份路径下。

  • 创建目录 ProjectData~，创建文件 package.json。

  • 将Assets 、 Packages 、 ProjectSettings 三个目录移动到 ProjectData~ 目录下。

  • 删除 ProjectSettings 目录中的文件 ProjectVersion.txt （如果有）。

  • 编写 package.json 文件的内容。（参考文档包清单）

  • 在根目录创建一个目录 package，然后将项目根目录内除此之外的其它所有内容都移动到其中，让其作为根目录。最终目录结构如下：

  • 通过压缩软件或 CMD（Windows系统）将 package 目录压缩为一个 .tgz 格式的压缩包文件，压缩包名称可以按照 Unity 内置模板格式 项目名称-模板版本号.tgz ，例如 com.unity.template.custom.empty-1.0.0.tgz。

  • 打开 Unity 编辑器安装目录，通常是 ..\Unity\Editor\版本号\Editor\Data\Resources\PackageManager\ProjectTemplates\，将我们的 .tgz 压缩包文件复制到该目录下。

  • 打开Unity Hub（如果已打开则关闭后重新打开），新建项目，选择自定义模板，测试新建的项目是否按照预期创建和运行。

注意：上述流程基于 Unity 2020.3.48f1 进行，如果其它版本存在不同流程，后续将视情况进行补充。

Q & A

Q: 为什么依照自定义模板创建的新项目还是会引入某些没有显示要求的依赖包？
A: 这是编辑器内置行为或 Unity Hub 预设行为，涉及到编辑器设置需要这些依赖包，你可以提前在模板中编写处理脚本，每次基于模板创建项目时将 Unity 额外添加的资源包移除。
