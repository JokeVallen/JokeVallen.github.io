Unity3D编辑器协程工具

更新于2026-03-13|Unity开发
|浏览量:

本工具由博主与 AI 共同开发，该版本为轻量版本，未附带过多功能，只覆盖了基础的功能。完全版本将另行发布，敬请期待。

前言

协程可以帮助我们在编辑器环境下执行异步任务。然而，Unity 的编辑器协程系统与运行时的协程系统有所不同，官方没有预留相关 API。为了满足这一需求，我们开发了一个专门用于 Unity 编辑器的轻量的协程工具。

功能特性

核心功能

  • ✅ 编辑器协程执行 - 在 EditorApplication.update 中运行协程，完全支持 yield return
  • ✅ 泛型返回值 - 通过 EditorCoroutine<T> 获取协程执行结果
  • ✅ 取消令牌支持 - 使用 EditorCoroutineCancelToken 优雅地中止协程
  • ✅ 嵌套协程 - 自动处理嵌套迭代器，支持复杂的协程链
  • ✅ 异常捕获 - 内置异常处理机制，不会导致编辑器崩溃
  • ✅ 常用的扩展方法 - WaitFrame、WaitSeconds、WaitUntil、Delay 等
  • ✅ 生命周期管理 - Start/Stop/Dispose 完整的资源管理

无依赖

  • 不依赖任何第三方库
  • 仅使用 Unity 编辑器公开的 API
  • 最小化代码体积

示例

请参考资源下载中的 example.unitypackage，其中包含了多个使用示例，展示了如何在编辑器中使用协程工具来执行异步任务。

  • RenameWindow：示例演示了批量给选定目录下的指定类型的资源进行随机命名的功能，涉及了协程处理批量数据、嵌套协程、协程取消的使用。
  • DownloadWindow：示例演示了从指定网址下载图片资源保存在本地并在窗口显示的功能，涉及了嵌套协程、多协程处理的使用。
  • AnalyticWindow：示例演示了分析项目Assets目录下所选类型资源的数量的功能，涉及了带返回值的协程、嵌套协程、多协程处理的使用。

Unity常识篇（一）
Awake 与 OnDestroy 的关系当组件被添加到活动状态的游戏对象时，Awake 会被调用，当组件被销毁时，OnDestroy 会被调用。如果 Awake 未被调用，那么 OnDestroy 也不会被调用，例如将组件添加到禁用状态的游戏对象然后再销毁组件时，二者都不会被调用。
