Unity编辑器IMGUI扩展（一）

更新于2026-04-30|Unity开发
|浏览量:

前言

在基于 IMGUI 进行 Unity 编辑器扩展时会涉及对一些常见绘制逻辑的封装，其中 EditorGUI.ObjectField 和 EditorGUILayout.ObjectField 则是本文的主题，我们没有直接封装原始的 ObjectField API，而是仿照原始 API 的功能和样式进行重构，但重构不会覆盖官方 API 的功能而是扩展，扩展方向分为无选择器版本和自定义选择器按钮点击行为版本，针对扩展代码进行了单元测试和功能测试。

API

本节列出 EditorObjectFieldUtility 对外公开的所有方法。每个方法都提供了泛型和非泛型两个版本，并支持 Rect（手动布局）和 Layout（自动布局）两种模式。

1. 无选择器按钮的对象字段

此类方法不显示对象选择器按钮（即右侧小圆点），但保留拖拽赋值和点击定位功能。

NoPickerObjectField<T>

  • 参数
    • rect : 控件所在的屏幕矩形。
    • label : 显示在控件左侧的标签（null 表示无标签）。
    • value : 当前对象引用。
    • allowSceneObject : 是否允许拖放场景中的对象（仅对非持久化资源有效）。
  • 返回 : 类型为 T 的对象（可能被拖放修改）。
  • 备注 : 内部直接调用 NoPickerObjectFieldInternal。

NoPickerObjectField (非泛型)

  • 参数 : 额外传递 objectType 限制可接受的对象类型。
  • 返回 : UnityEngine.Object 类型，使用时可进行类型转换。
  • 备注 : 若传入的 value 类型不符合 objectType，则自动清空为 null。

Layout 版本

  • 功能与 Rect 版本完全相同，但通过 EditorGUILayout.GetControlRect 自动获取布局矩形，适用于 Editor.OnInspectorGUI 等自动布局上下文。

2. 带自定义选择器按钮的对象字段

此类方法在控件右侧保留一个与官方样式一致的按钮，点击时触发自定义回调，不弹出默认的对象选择器。可通过回调自行实现选择逻辑。

ObjectField<T>

  • 参数
    • onPickerClick : 当右侧按钮被点击时执行的回调，传入当前字段的值（可能为 null）。
  • 其他 : 同 NoPickerObjectField。

ObjectField (非泛型)

  • 与泛型版类似，但回调参数类型为 UnityEngine.Object。

Layout 版本

  • 在自动布局中提供相同功能。

行为总结

功能 NoPicker 系列 ObjectField 系列

拖拽赋值 ✓ ✓
类型过滤 ✓ ✓

点击字段定位 (Ping) ✓ ✓

总结

本文介绍的 EditorObjectFieldUtility 是对 Unity 官方 ObjectField 的轻量级扩展，适合以下场景：

  • 需要完全禁止对象选择器，只通过拖拽或代码赋值的方式设置引用。
  • 需要自定义拾取器按钮的行为，例如打开一个定制的搜索窗口、调用 EditorGUIUtility.ShowObjectPicker 进行更复杂的筛选，或直接显示当前对象信息。
  • 希望在自动布局（EditorGUILayout）中方便地使用上述功能，同时保持与官方 API 相似的调用方式和视觉风格。

通过精简 API 重载（统一使用 GUIContent 传递标签），我们消除了 null 参数引发的编译二义性，并增加了 Event.current 的健壮性处理，确保即使在单元测试等无 GUI 上下文的环境下也不会抛出异常。

同时，配套的单元测试和手动测试窗口保证了该工具类的可靠性与易用性。
